# 🧾 产品需求文档（PRD）

## 1. 基本信息

**产品名称（暂定）**
Chinese Name Studio

**版本**
MVP v0.1

**产品类型**
Web 应用（Next.js）

**目标用户**
非中文母语用户（主要是英语用户）

---

## 2. 产品背景

随着 AI 工具普及，用户对“个性化内容生成”需求增加。
现有中文名生成工具大多：

* 随机生成
* 无文化解释
* 不与用户个性绑定

本产品希望提供：

👉 **基于用户个性与表达需求，生成有文化意义、自然、可信的中文名**

---

## 3. 产品目标

### 3.1 MVP 核心目标

验证以下假设：

> 用户是否愿意输入个人信息，并认可 AI 生成的中文名是“有意义且适合自己”的。

---

### 3.2 成功指标（MVP）

**核心指标：**

* 名字生成完成率 ≥ 60%
* 用户点击“重新生成”比例 ≤ 40%（越低说明满意度越高）

**辅助指标：**

* 页面停留时间 ≥ 60 秒
* 返回率（同一用户再次使用）≥ 15%

---

## 4. 用户需求

### 4.1 用户核心需求

1. 想要一个“像真正中文名字”的名字
2. 名字要和自己的性格/气质匹配
3. 名字要有解释，可以讲给别人听
4. 不希望名字显得奇怪或不自然

---

### 4.2 用户痛点

| 痛点   | 描述          |
| ---- | ----------- |
| 随机生成 | 名字没有意义      |
| 不自然  | 像拼接字，不像真实名字 |
| 无解释  | 不知道为什么这样取   |
| 不敢用  | 担心文化错误      |

---

## 5. 产品范围（MVP）

### 5.1 包含功能

* 用户输入个人信息
* AI 生成中文名（1 主 + 2 备选）
* 每个名字附带解释
* 支持重新生成

---

### 5.2 不包含（MVP 不做）

* 用户登录系统
* 收藏/历史记录
* 发音功能
* 分享图片生成
* 多语言支持
* 命理/五行系统
* 名字评分系统

---

## 6. 核心流程

### 6.1 用户流程

```
进入首页
→ 点击 CTA
→ 填写信息表单
→ 提交
→ 等待生成（加载态）
→ 查看结果
→ 可选择重新生成
```

---

## 8. 核心逻辑（AI 生成）

---

## 8.1 输入结构

```json
{
  "englishName": "...",
  "desiredVibe": "...",
  "about": "...",
  "useCase": "..."
}
```

---

## 8.2 生成流程（逻辑要求）

AI 必须执行以下步骤（逻辑约束，不一定展示）：

1. 理解用户个性与目标气质
2. 确定命名风格（自然 / 优雅 / 专业等）
3. 选择合适的姓氏
4. 组合名字（避免随机）
5. 验证：

   * 是否像真实中文名
   * 是否易读
   * 是否符合语境

---

## 8.3 输出要求

必须满足：

* 输出 1 个主推荐 + 2 个备选
* 每个名字必须：

  * 自然
  * 常见汉字
  * 无歧义或负面含义
* 必须提供解释

---

## 8.4 生成规则（关键）

必须在 prompt 中明确：

* 禁止随机字符组合
* 禁止使用极生僻字
* 禁止“小说风/过度中二”名字
* 优先真实中国人会使用的名字
* 名字必须与用户输入有关

---

## 9. 非功能需求

---

* 页面加载 ≤ 2 秒

---

## 9.2 稳定性

* API 错误率 ≤ 2%
* 超时需返回 fallback 文案

---

## 9.3 可用性

* 支持桌面端优先
* 移动端基础适配

---

## 10. 风险与约束

---

### 10.1 风险

| 风险       | 描述     |
| -------- | ------ |
| AI 生成不自然 | 用户失去信任 |
| 名字文化错误   | 用户不敢使用 |
| 结果太“像模板” | 失去产品差异 |

---

### 10.2 对策

* 强 prompt 约束
* 限制输出数量（避免质量下降）
* 强化解释部分

---

# ✅ MVP 一句话总结

一个基于 Next.js 的 AI 网站，通过用户输入的个性与需求，像“中文命名顾问”一样生成 1 个主推荐中文名和 2 个备选，并解释名字为何适合用户。
